# 工程日志：CCS 调试控制台快速观察模式

日期：2026-06-11

## 背景

本次工作围绕 `C:\ti\ccs1280` 下的 CCS 12.8.0、F28034 工程 `8065_d9_board_test_ccs` 和 XDS100v2 调试链路展开。目标是让网页控制台不只显示构建/日志，还能在 RAM 调试时观察程序运行状态、变量和寄存器。

## 本次新增能力

1. 修复 DSS 失败误报成功。
   - 当 `dss.bat` 进程返回码为 0，但 stdout/stderr 中出现 `SEVERE`、`Exception`、`Error connecting` 等 DebugServer 失败标志时，工具会把 `exit_code` 改成非 0。
   - 对目标无响应、held in reset、连接目标失败给出诊断提示。

2. 新增变量/寄存器读取。
   - 支持读取 `PC`、`SP`、`GpioDataRegs.GPADAT.all`、`GpioCtrlRegs.GPADIR.all`、`ECanaRegs.CANES.all` 等只读表达式。
   - 读取时会加载 `.out` 符号表；优先使用 RAM `.out`，没有 RAM `.out` 时可只加载 FLASH `.out` 的符号，不执行 Flash 烧录。
   - 限制表达式格式，拒绝 `PC = ...`、`;`、函数调用等可能写目标或执行 GEL 的输入。

3. 中文化网页控制台。
   - 将按钮、状态、摘要、日志入口改成中文。
   - 读取值在摘要区显示为简洁格式，例如 `GpioDataRegs.GPADAT.all = 0xFE112200 (4262428672)`。
   - 原始 JSON/DSS 输出保留在“原始日志”折叠区。

4. 新增观察模式。
   - 支持 Watch 列表，默认包含 `PC`、`SP`、`GpioDataRegs.GPADAT.all`。
   - 第一版逐项读取后发现每个表达式单独启动 DSS 约需 11-13 秒。
   - 随后改为批量读取，一次 DSS 脚本读取整组 Watch 表达式。

5. 新增常驻快速调试会话。
   - `LiveDebugSession` 启动一个常驻 DSS 进程和本地 socket 服务。
   - Web API 通过 `/api/live/start`、`/api/live/command`、`/api/live/stop` 控制常驻会话。
   - `开始观察` 会启动快速会话；快速会话只加载符号表，不重新下载程序，避免打断已经运行的 RAM 程序。
   - 后续 Watch、Run、Halt、Reset 优先走常驻会话，避免反复启动 DSS。

## 安全边界

- `加载 RAM` 只允许 RAM `.out`，仍拒绝 FLASH `.out` 的 `loadProgram()`。
- 表达式读取默认只读，拒绝赋值和函数调用。
- 快速会话启动时只加载符号表，不重新加载程序；用户应先执行 `加载 RAM -> 运行 -> 开始观察`。
- 本工具仍不做 Flash 擦写/烧录。

## 现场观察

已从日志确认以下运行中读取结果：

| 表达式 | 示例值 | 旧方式耗时 |
|---|---:|---:|
| `PC` | `0x3FF8A1` | 约 11.5 s |
| `SP` | `0x0400` | 约 12.1 s |
| `GpioDataRegs.GPADAT.all` | `4262428672` | 约 12.8 s |

旧方式慢的根因是每个变量都重新启动 DSS。当前版本通过批量读取和常驻会话降低该开销。

## 已知限制

- 运行中实时读变量是否成功，仍取决于 C2000/XDS/DSS 对目标运行状态下表达式求值的支持。
- 局部变量只有在 CPU 暂停于对应函数作用域内才可靠。
- 最适合持续观察的是全局变量、静态变量、CPU 寄存器和外设寄存器结构体。
- 常驻 DSS 会话会占用 XDS100v2；使用后应点击“退出快速会话”释放。

## 推荐操作顺序

1. 打开 `http://127.0.0.1:8766`。
2. 选择 `8065_d9_board_test_ccs`。
3. 点击 `加载 RAM`。
4. 点击 `运行`。
5. 在 Watch 列表加入需要观察的表达式。
6. 点击 `开始观察`。
7. 调试结束后点击 `退出快速会话`。
